물론입니다. 선생님을 위한 AI 기반 교육 자료 앱 제작에 활용할 수 있는 상세한 프롬프트(기획서)를 생성해 드릴게요. 이 프롬프트는 개발자나 기획자가 앱의 목표, 기능, 기술 사양을 명확히 이해하는 데 도움을 줄 수 있습니다.

교사용 AI 교육 비서 앱 개발 프롬프트
1. 프로젝트 목표 (Project Goal)
교사가 수업 및 연구 자료를 손쉽게 관리하고, AI 기술을 활용하여 자료를 다각적으로 분석 및 가공할 수 있는 지능형 교육 지원 플랫폼을 개발한다. 이를 통해 교사의 수업 준비 시간을 단축하고 교육의 질을 향상시키며, 학생들에게는 맞춤형 학습 경험을 제공하는 것을 목표로 한다.

2. 핵심 사용자 (User Personas)
교사 (Teacher):

자신만의 수업 자료(문서, PDF, 웹 링크 등)를 업로드하고 관리하고 싶어 한다.

AI를 통해 자료를 요약하고, 핵심 개념을 추출하고 싶어 한다.

학생들의 예습을 돕기 위한 학습 가이드나 퀴즈를 자동으로 생성하고 싶어 한다.

다양한 언어 배경을 가진 학생들을 위해 다국어 자료를 생성하고 싶어 한다.

자신의 학생들을 관리하고, 생성된 자료를 쉽게 공유하고 싶어 한다.

학생 (Student):

교사가 공유한 학습 자료, 요약본, 퀴즈 등을 통해 수업을 예습/복습한다.

음성 자료(팟캐스트)를 통해 이동 중에도 학습하기를 원한다.

3. 주요 기능 명세 (Key Features)
가. 사용자 인증 및 관리
교사 로그인: 이메일, Google 계정을 통한 소셜 로그인 기능 제공.

개인 자료실 (My Library): 로그인한 교사별로 독립된 클라우드 저장 공간 제공. 본인이 업로드한 자료만 열람 및 관리가 가능해야 함.

학생 관리:

교사가 본인의 학생을 이메일 주소로 직접 추가/삭제하는 기능.

[핵심 연동] Google Workspace 계정 연동을 통해 본인의 Google Classroom 클래스를 불러오고, 해당 클래스의 학생 명단을 동기화하는 기능.

나. 자료 업로드 및 관리
지원 파일 형식: PDF, DOCX, PPTX, TXT 등 텍스트 기반 문서 파일.

자료 관리: 폴더 생성, 파일 이름 변경, 삭제 등 기본적인 파일 관리 기능 제공.

다. AI 기반 콘텐츠 생성 (Core AI Functions)
업로드된 자료를 기반으로 다음 콘텐츠를 **다국어(한국어, 영어, 중국어, 태국어, 베트남어, 필리핀어)**로 생성한다.

1. AI 요약 (AI Summary):

자료의 핵심 내용을 간결하게 요약하여 제공.

요약 수준 조절: 간단한 요약(3줄), 상세 요약(문단별) 등 옵션 제공.

2. 예습 가이드 생성 (Pre-study Guide Generation):

주요 개념 추출: 자료에 등장하는 핵심 어휘와 그에 대한 간략한 설명.

학습 목표 제시: 이 자료를 통해 학생이 무엇을 배울 수 있는지 명시.

생각해 볼 질문: 학생의 사고력을 자극하는 심층 질문 3~5개 생성.

3. 퀴즈 생성 (Quiz Generation):

자료의 내용을 기반으로 퀴즈 문항 자동 생성.

문제 유형: 객관식, 참/거짓, 단답형 등 다양한 유형 지원.

정답 및 해설: 모든 문항에 대한 정답과 간단한 해설 자동 생성.

4. 팟캐스트 생성 (Podcast Generation):

업로드된 자료나 AI가 생성한 요약/가이드 텍스트를 기반으로 팟캐스트 대본을 생성.

생성된 대본을 자연스러운 **AI 음성(TTS)**으로 변환하여 음성 파일(MP3)로 제공.

다국어별 자연스러운 발음 지원.

라. 공유 및 활용
생성된 모든 콘텐츠(요약, 퀴즈, 팟캐스트)는 고유 링크로 생성되어 학생들에게 쉽게 공유 가능.

Google Classroom에 과제나 공지사항으로 바로 게시할 수 있는 'Classroom에 공유' 버튼 제공.

4. 기술 스택 제안 (Tech Stack Recommendation)
백엔드 (Backend): Python (Django/FastAPI)

이유: LangChain, Hugging Face Transformers 등 최신 AI/LLM 라이브러리와의 호환성이 뛰어나며, 데이터 처리 및 분석에 강점.

프론트엔드 (Frontend): React.js 또는 Vue.js

이유: 반응형 UI/UX 구축에 용이하며, 컴포넌트 기반 개발로 유지보수 효율성 높음.

데이터베이스 (Database): PostgreSQL 또는 MongoDB

이유: 사용자 정보, 파일 메타데이터, 생성된 콘텐츠 등 정형/비정형 데이터를 효율적으로 저장 및 관리.

AI/LLM 모델:

텍스트 생성/분석: Google의 Gemini API 또는 OpenAI의 GPT-4 API 활용. 다국어 처리 능력이 뛰어난 모델 선정 필수.

음성 변환(TTS): Google Cloud Text-to-Speech, Naver CLOVA Dubbing, ElevenLabs 등 고품질 다국어 음성 합성 API 활용.

클라우드/인프라 (Cloud/Infra): Google Cloud Platform (GCP) 또는 Amazon Web Services (AWS)

특히 GCP는 Google Workspace/Classroom 및 Gemini API와의 연동성 측면에서 유리.

5. 개발 로드맵 (Development Roadmap)
1단계 (MVP - 최소 기능 제품):

핵심 기능: 교사 로그인, 파일 업로드, 한국어/영어 2개 언어에 대한 AI 요약 및 퀴즈 생성 기능 구현.

사용자 관리: 이메일 기반의 간단한 학생 추가/삭제 기능.

2단계 (기능 확장):

다국어 지원: 모든 타겟 언어(중국어, 태국어, 베트남어, 필리핀어) 확장.

콘텐츠 확장: 예습 가이드 및 팟캐스트 생성 기능 추가.

연동 강화: Google Classroom 클래스 연동 기능 구현.

3단계 (고도화):

분석 대시보드: 학생들의 퀴즈 결과, 자료 열람 통계 등을 교사가 확인할 수 있는 대시보드 개발.

사용자 피드백: 생성된 콘텐츠에 대한 교사의 피드백을 받아 AI 모델을 미세 조정(Fine-tuning)하는 기능 도입.

모바일 앱: iOS/Android 네이티브 앱 또는 반응형 웹앱 고도화.
